Switch statements

Switch statements work similar to If statements, however, instead of testing for one condition, they are able to test for more than one condition. Switch statements begin with the keyword "switch," followed by a test value enclosed in parentheses. After this, we have a condition or case, followed by code block, also called an "action." If the test value matches the condition, the switch statement executes the action. If it does not match the condition, it moves on to the next switch case.

Break is a keyword in PowerShell which you can use to make it exit the control block it is currently executing.

Default case

What if we have some code we would like to execute when none of the cases? We can define a default case using the default keyword for our case. Here we have defined a default case that will execute when our input doesn't match at the end of the switch statement example we have been using so far. Since 3 does not match either of the defined cases, the action following the default case executes and PowerShell writes "Default case" to the console Window.

What would happen if we were to leave out the break statement in our code blocks? Instead of exiting out of the switch statement, PowerShell will move onto other cases to test if the condition matches. Let me show you what I mean.

On line 1 of the above code example, we have a switch statement with the test value of 1. PowerShell tests the value of the first case and finds a match. It executes the code on line 5 and writes "Case 1" to the console. We said that break statements are used to exit out of control blocks. Since we do not have a break statement, PowerShell moves on to our second case and finds that the test value does not match. It does not execute the action and moves on to the third case. Our third case tests for the value 1 again. Since it is a match, the following action will be executed and PowerShell will write "Case 3" to the console window. Since there is no break statement in this action, PowerShell moves on to the next case. Since the first and third cases matched, the action following the default case is skipped. PowerShell then reaches the end of the control block and exits out of the script.
